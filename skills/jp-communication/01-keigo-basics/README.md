# 01 · 敬语基础（エンジニア向け実用敬語）

> **目标**：掌握 Uchi/Soto（内/外）规则，在客户面前正确使用敬语  
> **核心模式**：内 vs 外 (Uchi vs Soto)  
> **难度**：中级

---

## 为什么重要 (Why This Matters)

你有没有在客户面前说过这样的话？

> 「山本部長もご出席されます。」
> （山本部长也会出席。）

这句话**语法正确，但场景错误**。

在日本商务场合，你刚刚犯了一个严重的敬语错误——用尊敬语称呼自己的上司，等于把自己公司的人抬得比客户还高。

客户心里可能在想：

> 「え？自分の上司に敬語？この人、ビジネスマナー分かってないな...」
> （对自己上司用敬语？这人不懂商务礼仪啊...）

**一个敬语错误，可能让你失去参加客户会议的机会。**

---

## 文化模式解析 (Cultural Pattern)

### 内 vs 外 (Uchi vs Soto)

这是商务敬语**最核心**的规则：

| 概念 | 日语 | 含义 | 敬语策略 |
|------|------|------|----------|
| **内 (Uchi)** | うち | 自己人（本公司） | 使用谦让语或普通形式 |
| **外 (Soto)** | そと | 外人（客户、合作方） | 使用尊敬语 |

### 关键规则

**当你与 Soto（外部人员）交流时：**

1. 对 **Soto** 使用尊敬语（抬高对方）
2. 对 **Uchi**（包括你的上司、CEO）使用谦让语或普通形式（压低己方）

### 为什么会这样？

这不是对上司不敬，而是**集体主义的体现**：

- 在客户面前，整个公司是一个"内部集体"
- 抬高自己公司的人 = 自夸 = 失礼
- 压低自己公司的人 = 谦虚 = 得体

**记住**：在客户面前，你的 CEO 也是"自己人"，不用敬语。

## 实用模板 (Template)

### Uchi/Soto 邮件检查清单

给客户发邮件前，按此清单检查：

```
[ ] 1. 识别 Soto（外部）：客户是谁？
       → 本例：XYZ公司的佐藤様

[ ] 2. 识别 Uchi（内部）：提到哪些自己人？
       → 本例：我、我的上司山本部长

[ ] 3. 对 Soto 使用尊敬语：
       - いらっしゃいます（在/来）
       - ご覧になる（看）
       - おっしゃる（说）

[ ] 4. 对 Uchi 使用谦让语或普通形式：
       - おります（在）
       - 拝見する（看）
       - 申し上げる（说）

[ ] 5. ⚠️ 关键检查：提到上司时
       ✗ 山本部長 → 用职称称呼 = 抬高
       ✓ 部長の山本 / 担当の山本 → 普通称呼 = 正确
```

### 常用敬语对照表

| 动作 | 普通形式 | 尊敬语（对 Soto） | 谦让语（对 Uchi） |
|------|----------|-------------------|-------------------|
| 在 | いる | いらっしゃる | おる |
| 说 | 言う | おっしゃる | 申す/申し上げる |
| 看 | 見る | ご覧になる | 拝見する |
| 做 | する | なさる | いたす |
| 去/来 | 行く/来る | いらっしゃる | 参る/伺う |

## 对比示例 (Bad → OK → Good)

**场景**：发邮件给客户，通知下次会议自己和上司会参加

### ❌ Bad Example

```
次回の打ち合わせには、山本部長も参加されます。
```

**问题分析**：
- `山本部長` — 用职称称呼自己上司 = 抬高
- `参加されます` — 对自己人用尊敬语 = 错误

**客户的感受**：「自分の上司を敬語で呼んでる...ビジネスマナーが分かってないな」

---

### 🔶 OK Example

```
次回の打ち合わせには、当社の山本も同席します。
```

**分析**：
- `当社の山本` — 正确（不用职称）
- `同席します` — 普通敬体，技术上正确
- 但整体有点生硬，缺乏温度

**客户的感受**：「まあ、正しいけど...ちょっと機械的だな」

---

### ✅ Good Example

```
次回のお打ち合わせですが、
弊社からはわたくしLiと、担当部長の山本が
同席させていただきます。

ご不明な点がございましたら、お気軽にお申し付けください。
```

**为什么这个版本好？**

| 要素 | 体现 |
|------|------|
| `お打ち合わせ` | 加「お」表示对会议的尊重 |
| `弊社` | 谦称（我司）vs 御社/貴社（贵司） |
| `わたくし` | 正式场合的"我" vs 私（わたし） |
| `担当部長の山本` | 正确：职务说明 + 名字，不加敬称 |
| `同席させていただきます` | 谦让形式："承蒙允许参加" |
| `お申し付けください` | 尊敬语："请吩咐" |

**客户的感受**：「この人、ちゃんとしてるな。信頼できる。」

## 角色扮演场景 (5-min Role-play)

### 情景设定

你在和客户佐藤さん进行视频会议。你的上司山本部長也在线上，但正在静音状态。

佐藤さん突然问你：

> 「山本部長は新しいデザイン案、もうご覧になりましたか？」
> （山本部长看过新的设计方案了吗？）

你知道他已经看过了。

### 你的任务

用正确的 Uchi/Soto 敬语回答这个问题。

**注意**：客户用了尊敬语称呼你的上司（因为对客户来说，山本部长是"他人"）。但你回答时，必须用谦让语（因为对你来说，山本部长是"自己人"）。

### 错误回答 ❌

```
はい、山本部長はご覧になりました。
```

（错：对自己人用了尊敬语）

### 正确回答 ✓

```
はい、部長の山本はすでに拝見しております。
先日のご提案について、大変興味深く拝見したとのことです。
```

**要点**：
- `部長の山本` — 不用「山本部長」
- `拝見しております` — 谦让语（看→拝見する）
- 还可以补充上司的正面反馈，帮助推进沟通

## 失误恢复 (Recovery Line)

### 场景：在客户面前用错了敬语

你意识到刚才对上司用了尊敬语。

### 日语恢复用语

```
失礼いたしました。
身内に対して敬語を使ってしまいました。

正しくは、「部長の山本はすでに確認しております」でした。
大変申し訳ございません。
```

### 解释

| 部分 | 作用 |
|------|------|
| 失礼いたしました | 承认失礼 |
| 身内に対して敬語を... | 明确说明错误类型 |
| 正しくは〜 | 主动更正 |
| 大変申し訳ございません | 郑重道歉 |

**要点**：日本人欣赏的是**立即意识到错误并更正**的态度。自我更正比假装没发生好得多。

## 关东 vs 关西 (Regional Nuance)

### Uchi/Soto 规则：全国通用，无地区差异

这是商务日语的基本规则，关东关西都一样严格。

### 唯一差异：关西客户可能更快放松

关西的客户可能更早说：

```
そんな丁寧じゃなくてもええよ。
```
（不用那么客气啦。）

**但是**：在关系真正熟络之前，继续使用正确敬语。这是展示专业性的机会。

---

## 职业进阶 (Bamboo Ceiling Breaker)

### 从"外国工程师"到"商务专业人士"

很多优秀的中国工程师被卡在技术岗位，无法转型为：
- 解决方案架构师 (Solution Architect)
- 技术主管 (Tech Lead)
- 售前工程师 (Pre-sales Engineer)

原因之一：**无法被信任出现在客户面前**。

### Uchi/Soto 的职业价值

| 掌握前 | 掌握后 |
|--------|--------|
| "技术不错但不太懂规矩" | "既专业又得体" |
| 客户会议只能旁听 | 可以代表公司发言 |
| 晋升受限于技术路线 | 管理/客户路线打开 |

### 信任信号

正确使用 Uchi/Soto 敬语，向客户和管理层发出的信号是：

> 「この人は日本のビジネス文化を理解している。
> 　お客様の前に出しても大丈夫だ。」
> （这个人理解日本商务文化，可以让他面对客户。）

这是突破"竹の天井"的关键钥匙之一。
