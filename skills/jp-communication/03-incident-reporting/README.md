# 03 · 障害報告の日本語 — 坏消息如何传达

> **目标**：学会在危机情况下冷静、专业地传达坏消息  
> **核心模式**：事実と意見を分ける (Jijitsu to Iken wo Wakeru)  
> **难度**：高级

---

## 为什么重要 (Why This Matters)

凌晨 3 点，你收到告警：生产环境挂了。

上司打来电话："怎么回事？"

你慌了，脱口而出：

> 「たぶん田中さんのコードにバグがあったせいです！全部壊れました！」
> （大概是田中的代码有 bug！全崩了！）

恭喜，你刚刚：
- 在没有证据的情况下指责同事
- 说了"大概"表示你在猜测
- 用"全崩了"制造恐慌

**这种沟通方式会让你失去信任。**

在日本职场，危机沟通的核心原则是：

> **事実と意見を分ける** — 分离事实与意见

你应该报告的是：**发生了什么、影响是什么、正在做什么**。

而不是：**谁的错、可能是什么原因、情况有多糟糕**。

---

## 文化模式解析 (Cultural Pattern)

### 事実と意見を分ける

| 日语 | 含义 |
|------|------|
| 事実 (Jijitsu) | 事实 — 可观察、可验证的客观信息 |
| 意見 (Iken) | 意见 — 主观判断、推测、情绪 |

### 危机沟通的三原则

| 原则 | 做法 |
|------|------|
| **只说事实** | 可观察的现象，不加推测 |
| **不找犯人** | 现在不是追责的时候 |
| **保持冷静** | 专业的口吻，不制造恐慌 |

### 犯人探し（はんにんさがし）= 禁忌

```
犯人探し = 找谁的错 = 绝对禁止
```

在事件进行中追责是：
- 浪费时间
- 制造对立
- 不专业的表现

事后可以做 RCA（根因分析），但现在的优先级是：**止血**。

## 实用模板 (Template)

### 5 行障害报告模板（第一报）

事件发生时，立即发送的 Slack/邮件：

```
件名：【障害報告・第一報】 XXシステム YY障害

CC: @dev-team, @support-team, @manager

1. 発生時刻：HH:MM JST

2. 事象（FACT）：
   [现象描述 — 可观察的事实]
   例：ユーザーがログインしようとすると500エラーが発生する

3. 影響範囲：
   [谁受影响、影响程度]
   例：全ユーザーがログイン不可。推定影響ユーザー数：約1,000名

4. 現在の対応：
   [现在正在做什么]
   例：担当者（Li, Sato）にて原因調査中

5. 次回の報告：
   [下次更新时间]
   例：15分後、HH:MMに第二報を報告します
```

### 后续报告模板（第二报以后）

```
件名：【障害報告・第二報】 XXシステム YY障害

1. 現在の状況：
   [恢复了/还在调查/已定位原因]

2. 原因（判明している場合）：
   [如果已确定原因]

3. 対応状況：
   [已采取的措施]

4. 復旧見込み：
   [预计恢复时间]

5. 次回の報告：
   [下次更新时间]
```

## 对比示例 (Bad → OK → Good)

**场景**：你是 On-call 工程师。部署后健康检查失败，服务自动回滚。

### ❌ Bad Example：指责 + 猜测 + 恐慌

```
デプロイが失敗しました！
田中さんのコードにバグがあったせいです。
全部壊れました！どうしよう！
```

**问题分析**：

| 问题 | 表现 |
|------|------|
| 指责他人 | 「田中さんのコードにバグがあったせい」 |
| 没有证据 | 只是猜测，还没调查 |
| 制造恐慌 | 「全部壊れました！どうしよう！」 |
| 缺乏信息 | 没说影响范围、当前对应 |

**上司的感受**：「この人、パニックになってる。任せられないな...」

---

### 🔶 OK Example：只有事实，但不够完整

```
v1.2のデプロイが失敗しました。
今からロールバックします。
```

**分析**：
- 没有指责 ✓
- 说明了对应 ✓
- 但缺少影响范围、时间线、下次更新

**上司的感受**：「まあ、冷静だけど、もう少し情報がほしいな」

---

### ✅ Good Example：专业的障害报告

```
【障害報告】デプロイ失敗のご報告です。

1. 発生時刻：15:00 JST

2. 事象：
   本番デプロイ(v1.2)がヘルスチェックに失敗し、
   自動的にv1.1へロールバックされました。

3. 影響範囲：
   現在、サービスは正常稼働中です(v1.1)。
   ユーザーへの影響はありません。

4. 現在の対応：
   リリースログを確認中です。
   XX処理に問題があった可能性が高いと見ています。

5. 次回の報告：
   30分後、15:30に原因調査の結果を報告いたします。
```

**为什么这个版本好？**

| 要素 | 体现 |
|------|------|
| 时间线清晰 | 发生时刻 15:00 |
| 只说事实 | 健康检查失败 → 自动回滚 |
| 影响明确 | 服务正常，无用户影响 |
| 有推测也标注 | 「可能性が高いと見ています」（明确是推测） |
| 有下次更新 | 30 分钟后报告 |

**上司的感受**：「冷静に対応してるな。任せておいて大丈夫だ。」

## 角色扮演场景 (5-min Role-play)

### 情景设定

你是 On-call 工程师。凌晨 2 点收到告警：

```
[ALERT] CPU 100% on batch-server-01
```

这意味着夜间批处理可能延迟，客户的每日报告会晚到。

根因还不清楚。

山本部長打来电话："发生了什么？"

### 你的任务

进行一个冷静、事实导向的口头汇报。

**要点**：
- 只说你**确定知道**的事实
- 说明影响
- 说明你**正在做**什么
- 不要说"可能是"、"大概"、"我觉得"

### 参考对话

```
山本：もしもし、どうした？

你：お疲れ様です、Liです。
    夜間バッチサーバーで障害が発生しました。
    ご報告いたします。

山本：状況は？

你：2:00にアラートを受信しました。
    batch-server-01のCPU使用率が100%に達しています。

    これにより、夜間バッチ処理が遅延する可能性があります。
    対象は、翌朝のレポート配信約500社です。

山本：原因は分かった？

你：現在調査中です。
    サーバーにログインして、プロセスを確認しています。

    10分後に第二報をお送りします。

山本：分かった。よろしく頼む。

你：承知しました。失礼します。
```

**要点**：
- 自报家门
- 时间 + 现象 + 影响
- 承认还在调查（不瞎猜）
- 承诺下次更新

## 失误恢复 (Recovery Line)

### 场景：你刚才说了情绪化的话或猜测性内容

上司皱眉了，你意识到自己失态了。

### 日语恢复用语

```
申し訳ございません、
少し混乱しており、憶測で話してしまいました。

事実は〜〜です。

現在、対応を進めております。
10分後に改めてご報告いたします。
```

### 解释

| 部分 | 作用 |
|------|------|
| 混乱しており | 承认自己慌了（可以理解的人之常情） |
| 憶測で話してしまいました | 承认说了猜测的话 |
| 事実は〜〜です | 重新说一遍**确定的事实** |
| 改めてご報告 | 承诺会正式更新 |

**要点**：承认失态 → 纠正 → 回到专业模式

## 关东 vs 关西 (Regional Nuance)

### 危机沟通：全国统一标准

这是少数没有地区差异的场景。无论关东还是关西，专业的障害报告都要求：
- 冷静
- 事实导向
- 不追责

### 唯一差异：上司的反应风格

| 地区 | 上司可能的反应 |
|------|----------------|
| **关东** | 「分かった。引き続き対応よろしく。」（冷静） |
| **关西** | 「あー、やってもうたか。しゃあない、切り替えていこか。」（口语） |

关西上司可能用更随意的语气，但**对你的期待是一样的**：专业的报告。

## 职业进阶 (Bamboo Ceiling Breaker)

### 危机中见真章

任何人都能在顺风顺水时表现良好。

但当系统崩溃、半夜告警、客户投诉时——**这才是展示真正实力的时刻**。

### 危机沟通的职业价值

| 表现 | 上司的评价 |
|------|------------|
| 慌张、指责、猜测 | 「この人は危機対応に向いてない」 |
| 冷静、事实、专业 | 「この人は頼りになる。安心して任せられる」 |

### "Safe Pair of Hands"（稳妥的人）

在日本职场，有一种极高的评价：

> 「あの人なら大丈夫」
> （那个人的话没问题的）

这不是说你不会犯错，而是说：

- 你遇到问题不会慌
- 你会冷静处理
- 你会及时汇报
- 你是可以信赖的

这种"稳妥"的印象是通往高级职位的必要条件。

### 危机 = 机会

每次生产事故都是一次"面试"：

- 处理得好 → 信任度 +10
- 处理得差 → 信任度 -10

危机沟通能力是**无法伪装**的。它只能通过实战练习获得。
