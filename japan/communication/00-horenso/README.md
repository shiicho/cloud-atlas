# 00 · 日本职场沟通概览（ホウレンソウ）

> **目标**：掌握 Horenso（报告·联络·相谈）的基本框架，建立日本职场信任的第一步  
> **核心模式**：報・連・相 (Hō-Ren-Sō)  
> **难度**：入门

---

## 为什么重要 (Why This Matters)

你是不是经常遇到这样的情况？

- 任务完成了，发个"完了"就继续下一个——结果上司不知道你在干什么
- 遇到问题自己扛着解决——结果错过了最佳求助时机
- 项目进展顺利，觉得没必要汇报——结果上司以为你在摸鱼

在日本职场，**沉默不是金，而是黑箱**。

管理者最怕的不是坏消息，而是"不知道发生了什么"。中国工程师常见的"自己扛着把事办了"反而会让日本上司焦虑：

> 「あいつ、今何やってるんだ？大丈夫か？」
> （那家伙现在在干什么？没问题吧？）

**Horenso 不是微观管理，而是信任的基础设施。**

---

## 文化模式解析 (Cultural Pattern)

### 報・連・相 (Hō-Ren-Sō)

| 要素 | 日语 | 含义 | 场景 |
|------|------|------|------|
| **報告** | ほうこく (Hōkoku) | 汇报结果/进展 | 任务完成、阶段性进展 |
| **連絡** | れんらく (Renraku) | 传递信息 | 会议时间变更、客户反馈 |
| **相談** | そうだん (Sōdan) | 商量/请教 | 遇到困难、需要决策支持 |

### 核心理念

Horenso 的本质是**让上司没有"意外"**：

- 好消息要报告 → 让上司知道项目在正轨
- 坏消息要报告 → 让上司有时间准备对策
- 不确定的事要相谈 → 让上司参与决策，分担责任

**记住**：在日本职场，"我以为你知道"是最糟糕的借口。

## 实用模板 (Template)

### 朝一报告模板（Daily Morning Report）

每天早上在 Slack/Teams 发送状态更新：

```
件名：【作業報告】_Name_ _YYYY/MM/DD_

おはようございます。本日の作業予定をご報告いたします。

1. 昨日からの進捗
   - XX機能のバックエンド実装完了 (JIRA-123)
   - YYバグの原因特定済み

2. 本日の予定
   - 午前：YYバグ修正 + 単体テスト
   - 午後：ZZ設計ドキュメント作成

3. 課題・相談事項
   - YYバグ修正でDBスキーマ変更が必要かもしれません
   - 着手前に10分ほどご相談させていただけますでしょうか

以上、よろしくお願いいたします。
```

### 使用说明

| 部分 | 目的 |
|------|------|
| 昨日からの進捗 | **報告** - 让上司知道完成了什么 |
| 本日の予定 | **連絡** - 让上司知道今天的计划 |
| 課題・相談事項 | **相談** - 提前预警，请求支援 |

## 对比示例 (Bad → OK → Good)

**场景**：你完成了一个登录功能的开发任务 (JIRA-123)

### ❌ Bad Example

```
タスク完了しました。
```

**问题分析**：
- 太简短，像机器人一样
- 没有上下文，上司不知道是哪个任务
- 没有提供下一步信息

**日本人的感受**：「え？何が完了したの？報告として成り立ってないよ...」

---

### 🔶 OK Example

```
田中さん、JIRA-123の実装が完了しました。次のタスクに進みます。
```

**分析**：
- 基本信息齐全
- 但缺乏细节，平淡无奇
- 上司还需要自己去查 JIRA 看详情

**日本人的感受**：「まあ、一応報告はしてくれてるけど...」

---

### ✅ Good Example

```
お疲れ様です。Liです。

ご依頼いただいていたログイン機能(JIRA-123)の実装が完了しましたので、
ご報告いたします。

関連する単体テストも完了しており、現在QAチームのレビュー待ちです。
次のタスクに着手しますが、QAからフィードバックがあれば
そちらを優先対応いたします。

取り急ぎご報告まで。
```

**为什么这个版本好？**

| 要素 | 体现 |
|------|------|
| 开头问候 | `お疲れ様です` — 职场基本礼仪 |
| 自报家门 | `Liです` — 让上司快速识别 |
| 任务关联 | `ご依頼いただいていた` — 说明这是上司交代的任务 |
| 详细状态 | 不只是"完了"，还说明了测试状态 |
| 主动性 | 表明会优先处理 QA 反馈 |
| 简洁结尾 | `取り急ぎご報告まで` — 表示这只是快速更新 |

**日本人的感受**：「この人、しっかりしてるな。安心して任せられる。」

## 角色扮演场景 (5-min Role-play)

### 情景设定

昨天，田中さん（你的上司）让你调查一个 API 响应慢的问题。

**你的发现**：
- 原因是 ORM 的 N+1 查询问题
- 修复本身很简单
- 但需要升级一个库的版本，可能有副作用

现在是早上，你需要向田中さん汇报。

### 你的任务

练习一个完整的 Horenso：
1. **報告**：调查结果（发现了什么）
2. **連絡**：计划（打算怎么修）
3. **相談**：请示（库升级的风险需要确认）

### 参考对话

```
你：田中さん、おはようございます。昨日ご依頼いただいた
    API遅延の件で、ご報告があります。
    少しお時間よろしいでしょうか。

田中：うん、どうぞ。

你：調査の結果、N+1クエリが原因であることが特定できました。
    修正自体は比較的シンプルなのですが、
    ORMライブラリのバージョンアップが必要になる可能性があります。

    バージョンアップに伴う影響範囲について、
    着手前にご相談させていただきたいのですが、
    10分ほどお時間いただけますでしょうか。

田中：なるほど。じゃあ、10時から少し話そうか。

你：ありがとうございます。10時に伺います。
```

## 失误恢复 (Recovery Line)

### 场景：忘记汇报某件事

上司问："那个 XX 的事情怎么样了？"而你忘记汇报了。

### 日语恢复用语

```
大変申し訳ございません。
私の報告が漏れておりました。

[补上内容]

今後は進捗があり次第、速やかにご報告いたします。
```

### 解释

| 部分 | 作用 |
|------|------|
| 大変申し訳ございません | 诚恳道歉（比すみません更正式） |
| 私の報告が漏れておりました | 承认是自己的疏忽（不找借口） |
| 今後は〜 | 承诺改进（展示学习态度） |

**要点**：日本职场重视的是**承认错误 + 承诺改进**的姿态，而不是解释原因。

## 关东 vs 关西 (Regional Nuance)

| 地区 | 风格 | Horenso 特点 |
|------|------|--------------|
| **关东（东京）** | 较为正式 | 书面化、文档驱动。朝一 Slack 报告是典型关东风格 |
| **关西（大阪）** | 较为口头 | 更喜欢面对面交流。上司可能更希望你直接走过来说 |

### 关西版本

在关西的公司，上司可能更喜欢这样：

```
田中さん、昨日の件なんですけど、ちょっといいですか？
```

（田中さん，关于昨天那事儿，现在方便吗？）

**核心**：Horenso 的*内容*是一样的，只是*媒介*不同。关西更偏好口头，关东更偏好书面。

## 职业进阶 (Bamboo Ceiling Breaker)

### 从"写代码的人"到"可信赖的专业人士"

很多中国工程师卡在"个人贡献者"的位置上，原因之一就是：

> 上司不知道你在干什么，所以不敢把重要的事交给你。

掌握 Horenso 能带来的改变：

| Before | After |
|--------|-------|
| 黑箱状态，上司焦虑 | 透明可见，上司安心 |
| 只能接执行性任务 | 可以接更复杂的项目 |
| 被动等待分配 | 主动沟通，展示主导力 |

### 晋升的第一步

日本公司在考虑晋升时，会问：

> 「この人に任せて大丈夫か？」
> （把事情交给这个人，没问题吗？）

答案取决于你平时建立的**信任资产**。

Horenso 是积累信任最基础、也是最有效的方式。它证明你不是一个需要被追着要进度的人，而是一个**主动管理自己工作域**的专业人士。
